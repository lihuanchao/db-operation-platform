# 复杂 SQL 表名提取的完美解决方案

## 问题描述
在数据库运维管理平台中，对于复杂的 SQL 查询（包含 CTE、子查询、多 JOIN 等），传统的 SQL 解析器（如 sqlparse）经常无法完整准确地提取所有表名，导致获取表结构和执行计划时出现错误或信息不全。

## 解决方案：混合 LLM 增强解析策略

### 核心思路
结合传统 SQL 解析和 LLM（大语言模型）的强大能力，形成优势互补：

1. **主要方法**：使用 LLM 解析 SQL，因为它能够理解复杂的 SQL 结构和上下文
2. **回退方案**：当 LLM 失败时，使用简单的正则表达式解析作为回退
3. **结果缓存**：对相同的 SQL 进行缓存，避免重复调用 LLM
4. **智能过滤**：对提取的表名进行验证和去重

### 方案架构图

```
输入 SQL
    │
    ├─ 检查缓存 ──是──▶ 返回缓存结果
    │
    ▼ 否
┌─────────────────────────────────────────┐
│         第一阶段：LLM 解析（主要）      │
│      - 理解 SQL 结构和上下文           │
│      - 识别真实表名，忽略 CTE 别名     │
│      - 处理复杂的子查询和视图          │
└─────────────────────────────────────────┘
    │
    ├─ 成功？ ──是──▶ 结果过滤与验证
    │
    ▼ 否
┌─────────────────────────────────────────┐
│      第二阶段：简单解析器（回退）      │
│      - 正则表达式提取                   │
│      - sqlparse 解析                    │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│         结果过滤与验证                  │
│      - 过滤 SQL 关键字                 │
│      - 去重（忽略大小写）              │
│      - 验证表名格式                    │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│         缓存结果                        │
│      - 存储 SQL → 表名映射            │
└─────────────────────────────────────────┘
    │
    ▼
返回最终表名列表
```

## 实现细节

### 1. LLM Prompt 设计

Prompt 的设计是成功的关键，需要：
- 明确说明需求（提取真实表名，忽略 CTE 别名）
- 提供多个示例
- 限制输出格式（只返回逗号分隔的列表）

```python
示例 Prompt：
"""请分析以下 SQL 语句，并提取其中所有引用的真实数据库表名（不包括临时表、CTE别名和子查询别名）：

SQL语句：
```{sql_text}```

要求：
1. 提取所有 FROM、JOIN、INNER JOIN、LEFT JOIN、RIGHT JOIN 等之后的表名
2. 如果是 CTE（WITH 子句），请忽略 CTE 的别名，只提取 CTE 内部 SQL 中引用的真实表名
3. 识别视图引用（视图应该被当作表名提取）
4. 绝对不要包括子查询别名和临时表
5. 输出格式：只返回逗号分隔的表名列表，不要添加任何其他内容、解释或说明

示例：
如果 SQL 是 "WITH active_users AS (SELECT * FROM users WHERE active = 1) SELECT * FROM active_users u JOIN orders o ON u.id = o.user_id"，
返回：users, orders
"""
```

### 2. 结果解析

LLM 可能返回各种格式的结果，需要处理：
- 逗号分隔列表
- 数组格式
- 换行分隔
- 带 markdown 代码块

### 3. 缓存策略

对相同的 SQL 进行归一化处理（去除多余空格），然后缓存结果，显著提高性能。

### 4. 表名验证与过滤

- 过滤 SQL 关键字（select、from、where 等）
- 验证表名格式（不含特殊字符）
- 去重（忽略大小写）

## 性能优化

1. **缓存命中**：相同 SQL 不会重复调用 LLM
2. **快速回退**：LLM 失败时快速回退到简单解析
3. **异步处理**：在批量优化场景中可以异步处理
