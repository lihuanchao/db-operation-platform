# 创建 SQL / ORM XML 优化任务页面 UI 优化设计

## 1. 背景

当前系统有两个任务创建页：

- `创建 SQL 优化任务`
- `创建 ORM XML 优化任务`

两页基础流程完整，已具备：

- 连接选择
- 数据库名输入
- SQL / XML 内容输入
- 表单校验
- 提交后跳转任务详情

但目前仍存在可优化点：

- 虽然功能一致，但两页在视觉细节上仍有轻微差异（不同渐变背景），一致性不够强
- 页面顶部与卡片内存在介绍性文案，信息密度偏高
- 结构是“可用”，但未形成统一的“创建任务工作区”体验
- 与已优化的“优化任务列表”页面风格衔接不够紧

本次优化目标是在不改业务行为的前提下，把两个创建页统一成同一套结构与视觉语言，并收敛介绍性内容。

## 2. 目标

### 2.1 功能目标

- 保持现有表单行为与提交逻辑不变
- 保持返回列表与创建后跳转详情逻辑不变
- 保持校验规则不变
- 统一两页结构与样式表达

### 2.2 体验目标

- 采用 `A 统一单卡片工作区` 方向
- 页面内容去介绍化，仅保留必要标题、字段、输入与操作
- 视觉风格与“优化任务列表”保持同一套浅色企业后台语言

### 2.3 非目标

- 不改 API / store / router
- 不引入新字段或新交互流程
- 不增加复杂引导或右侧摘要栏

## 3. 已确认方向

用户已确认：

- 优化范围：仅两个创建页
- 方案方向：`A 统一单卡片工作区`
- 文案策略：`去介绍化`

## 4. 总体方案

两个页面使用同一骨架：

1. 页面头部：标题 + 返回列表按钮
2. 主体卡片：任务配置表单
3. 输入区：SQL / XML 文本输入（两页唯一核心差异）
4. 操作区：取消 + 开始优化

两页只保留必要的业务文案，不再放解释性段落。最终体验是“进入即填写、填写即提交”的清晰工作流。

## 5. 页面结构设计

## 5.1 头部

保留：

- 页面标题
- 返回列表按钮

移除：

- 顶部介绍副标题（当前 p 文案）

目的：

- 降低视觉噪音
- 让用户直接进入配置动作

## 5.2 主体卡片

保留单卡片结构 `form-shell`，统一两页卡片风格：

- 白底
- 浅边框
- 轻阴影
- 圆角

卡片头仅保留 `任务配置` 标题，不增加描述。

## 5.3 表单区

保留现有字段与顺序：

- 数据库连接
- 数据库名
- SQL 内容 / MyBatis XML 内容

两页统一点：

- 字段布局
- 字体和间距
- 输入框风格
- 字符计数位置

两页差异点只保留在业务必要文案：

- 页面标题
- 文本域 label
- 文本域 placeholder

## 5.4 操作区

保留：

- 取消
- 开始优化

按钮行为不变，仍为右对齐操作区。

## 6. 文案与交互规范

## 6.1 去介绍化原则

删除介绍性文案，仅保留动作必要信息：

- 删除顶部副标题说明
- 不新增额外说明段落
- 不增加“如何填写”教程块

## 6.2 输入引导原则

保留 placeholder，但文案保持短句和直接表达，避免解释性长句。

## 6.3 交互行为边界

保持原有行为：

- `goBack` 返回列表
- `validate` 表单校验
- `createSqlTask` / `createMyBatisTask` 提交
- 成功后跳转详情页
- 失败提示不变

## 7. 视觉风格

## 7.1 统一风格

两个页面统一为同一视觉语言：

- 不使用不同方向或不同颜色的卡片渐变背景
- 统一卡片外观
- 统一按钮与边框色系

## 7.2 色彩约束

沿用系统当前配色主线：

- 主色：`#0369A1`
- 文本主色：`#0F2A3D`
- 边框浅色：`#D9E6F2`
- 背景浅色：延续当前全站背景

## 8. 响应式要求

- 桌面端保持当前字段并排布局（连接 / 库名）
- 移动端自动单列
- 操作按钮在小屏下保持可点击面积，不出现挤压失真

## 9. 实现边界

只修改以下文件：

- `frontend/src/views/OptimizationTaskCreateSql.vue`
- `frontend/src/views/OptimizationTaskCreateMyBatis.vue`

允许：

- 模板精简
- 样式统一
- 文案收敛

不允许：

- 改接口调用
- 改 store 方法签名
- 改路由路径

## 10. 验证与验收

至少验证：

- 两页可正常返回列表
- 两页必填校验不回归
- 两页提交成功后跳转详情不回归
- `npm test` 通过
- `npm run build` 通过
