# 权限管理页面 UI 优化设计

## 1. 背景

当前权限管理页已经具备基本功能：

- 左侧展示用户列表
- 右侧展示数据库连接授权
- 支持搜索用户
- 支持搜索连接
- 支持保存授权结果

但当前页面仍然更像“功能拼装页”，存在几个明显问题：

- 左右两栏虽然具备业务结构，但视觉层级弱，选中用户和授权对象的关系不够直观
- 用户列表仍然依赖表格阅读，页面重点不够聚焦
- 连接授权区域信息密度低，授权状态、已授权数量、操作结果不够可感知
- 空状态和搜索无结果状态较弱，用户只能通过上下文猜测当前处于什么状态
- 页面质感与登录页近期确立的“浅色、干净、专业、非装饰化”方向还未统一

本次优化目标是在不改后端接口和核心数据结构的前提下，把权限管理页升级成一个更清晰、更克制、更适合后台日常操作的双栏授权工作台。

## 2. 目标

### 2.1 功能目标

- 保留当前“左侧选用户，右侧分配连接”的主流程
- 让用户与授权对象之间的关系更清楚
- 增强页面状态反馈，但不引入复杂流程
- 提升搜索、选择、保存三个关键操作的可理解性

### 2.2 体验目标

- 风格偏“简洁质感优先”，而不是信息爆炸型后台
- 保持企业后台的专业感，避免花哨背景、重装饰和冗余说明
- 在桌面端保持双栏效率，在移动端自然退化为上下布局

### 2.3 非目标

- 不改后端授权模型
- 不新增接口
- 不引入复杂的批量授权流程或审批流
- 不做矩阵式权限总览

## 3. 用户已确认的方向

经过方案对比，本次采用以下设计方向：

- 布局方向：`A 双栏授权台`
- 视觉倾向：`简洁质感优先`
- 交互范围：`允许少量交互增强`

放弃的方向：

- 步骤式授权流：当前业务单一，过度引导会增加操作层级
- 矩阵式权限面板：更适合大规模权限盘点，不适合当前连接授权场景

## 4. 总体方案

权限管理页调整为一个清晰的双栏授权工作台：

- 左栏：用户选择面板
- 右栏：连接授权面板

页面的核心体验从“表格 + 复选框”转向“对象选择 + 授权卡片”，让用户能快速回答三个问题：

- 我当前正在给谁授权
- 这个人已经被授了多少项
- 我这次勾选的内容是什么

## 5. 页面结构设计

## 5.1 页面头部

保留当前标题区，但文案和层级更克制：

- 标题：`权限管理`
- 副标题：`为普通用户分配可访问的数据库连接`

头部不增加统计卡，不增加说明段落，保持页面开场简短。

## 5.2 左栏：用户选择面板

左栏从表格调整为更轻的选择列表，结构如下：

- 面板标题：`用户列表`
- 顶部搜索框：搜索姓名或工号
- 用户列表区域：按卡片式条目展示

每个用户条目展示：

- 姓名
- 工号
- 角色标签

交互规则：

- 普通用户可点击并切换当前选中状态
- 管理员用户仍可展示，但视觉上弱化，并在点击后提示“管理员默认拥有全部权限，无需单独授权”
- 当前选中用户使用明显边框、浅蓝底色和轻微阴影区分

视觉原则：

- 用户条目高度统一
- 标签色彩克制，不抢主流程
- 列表滚动区域独立，避免整页抖动

## 5.3 右栏：连接授权面板

右栏保留连接授权功能，但从“复选框直出”改为更完整的授权工作区：

- 面板标题：`连接授权`
- 顶部搜索框：搜索连接名或 IP
- 当前用户摘要区
- 连接授权列表
- 操作区

当前用户摘要区展示：

- 当前用户姓名与工号
- 已授权数量，例如：`已授权 3 项`

连接列表展示内容：

- 连接名称
- 管理 IP 或数据库主机
- 选中状态

交互呈现方式：

- 外层为可点击卡片
- 卡片内部保留复选能力
- 已选中项通过边框、背景和选中标记一起表达，而不是只靠复选框

## 6. 少量交互增强

本次允许的交互增强控制在“帮助理解和提效”范围内，不改变业务流。

### 6.1 已授权数量提示

在右栏顶部展示当前用户已授权数量。

目的：

- 让用户在保存前后能快速感知授权规模
- 避免必须人工数复选项

### 6.2 空状态优化

需要区分至少三类空状态：

- 未选择用户：提示先选择左侧普通用户
- 用户搜索无结果：提示未匹配到用户
- 连接搜索无结果：提示未匹配到连接

要求空状态文案明确，不使用泛化的“暂无数据”。

### 6.3 全选 / 清空

在右栏操作区增加：

- `全选当前结果`
- `清空当前结果`

规则：

- 仅作用于当前过滤后的连接结果
- 不影响搜索条件
- 不新增后端交互

### 6.4 保存按钮状态

保存按钮需要具备更明确的可操作反馈：

- 未选择普通用户时禁用
- 保存中显示 loading
- 放在右栏底部固定操作区，位置稳定

## 7. 视觉风格

页面风格统一到当前系统正在建立的浅色企业后台方向。

### 7.1 色彩

主色建议沿用现有蓝系：

- 主色：`#0369A1`
- 辅色：`#0EA5E9`
- 背景浅色：`#F3F7FB`
- 文本深色：`#0F2A3D`
- 边框浅色：`#D9E6F2`

原则：

- 不使用大面积渐变背景
- 不使用强对比装饰色块
- 选中态和主按钮保持统一蓝系语言

### 7.2 组件风格

- 面板卡片：白底、浅边框、轻阴影、圆角
- 用户条目卡片：比页面卡片更轻，强调选中态
- 连接授权项：清晰卡片化，但尺寸紧凑
- 搜索框：圆角，边框干净，聚焦态可见
- 主按钮：纯色或轻渐变蓝色均可，但整体以克制为主

### 7.3 排版

- 标题区保持短句
- 列表项信息分两层：主信息 + 次信息
- 避免一行内塞入过多字段

## 8. 响应式行为

桌面端：

- 左右双栏并排
- 左栏偏窄，右栏偏宽

平板与移动端：

- 自动切换为上下布局
- 左栏在上，右栏在下
- 搜索框和操作按钮宽度自适应
- 连接授权区允许内部滚动，不出现横向溢出

## 9. 可访问性与状态表达

页面需要满足以下基础要求：

- 可点击条目要有明显 hover 和 focus 态
- 选中状态不能只靠颜色表达
- 角色和状态建议同时用文字表达
- 空状态文案要清楚说明下一步动作

## 10. 实现边界

本次实现只改前端页面层：

- 主要修改文件：`frontend/src/views/PermissionManagement.vue`

允许的实现内容：

- 模板结构调整
- 页面级计算属性和轻量交互方法
- 样式重构

不做的内容：

- 不改 store 接口语义
- 不改后端 API
- 不引入新的权限模型

## 11. 测试与验证

至少完成以下验证：

- 现有权限选择与保存流程不回归
- 管理员点击提示逻辑保持正确
- 搜索过滤仍正常工作
- 全选 / 清空仅作用于当前过滤结果
- 桌面端与移动端布局均可用
- 前端测试和构建通过

## 12. 风险与注意事项

- 当前 `selectedConnectionIds` 直接绑定在 store 上，实现全选 / 清空时要避免误伤未过滤项
- 连接卡片化后，点击区域和复选组件事件要避免双触发
- 如果用户列表数据量后续增长，卡片式列表要保留可滚动和搜索能力

## 13. 结论

本次权限管理页优化采用“简洁质感优先”的双栏授权台方案，在不改变现有后端能力的前提下，通过以下方式提升体验：

- 更清楚的用户选中关系
- 更直观的连接授权对象表达
- 更明确的空状态和已授权反馈
- 适度提效但不过度复杂化的交互增强

该方案实现成本低，风险可控，且与当前系统正在统一的界面风格保持一致，适合作为当前阶段的权限管理页正式改版方案。
