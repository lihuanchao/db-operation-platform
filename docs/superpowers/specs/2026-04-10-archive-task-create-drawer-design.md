# 归档任务管理-新增任务页面 UI 优化设计（右侧抽屉）

## 1. 背景

当前“归档任务管理”页面在列表页内使用弹窗完成新增任务。现状可用，但对高频录入场景存在以下问题：

1. 表单结构层次感不强，信息组织偏平。
2. 关键录入动作（新增并立即执行）不够突出。
3. 交互反馈与页面上下文衔接一般，效率导向不够明确。

本次优化目标是在不改业务接口与核心数据结构的前提下，将“新增任务”升级为企业级、简洁高效的右侧抽屉工作区。

## 2. 目标与边界

### 2.1 目标

1. 新增入口改为右侧抽屉承载，提升与列表页上下文连续性。
2. 表单采用“核心信息区 + 高级配置区”分区。
3. 高级配置区默认展开（用户明确要求）。
4. 主操作明确为“保存并执行”。
5. 视觉保持企业级运维后台风格：克制、清晰、效率优先。

### 2.2 非目标

1. 不改后端 API 与字段含义。
2. 不新增任务字段，不改数据库模型。
3. 不引入多步骤向导，不改为独立路由页。

## 3. 已确认方案

用户已确认：

1. 容器形态：右侧抽屉。
2. 信息组织：D2（核心区 + 高级配置区）。
3. 高级配置：默认展开，不折叠。
4. 主按钮文案与行为：保存并执行。

## 4. 页面与交互设计

## 4.1 抽屉结构

1. 抽屉位置：右侧滑入。
2. 建议宽度：`560px`（在 520-600 区间内可微调）。
3. 标题：
   - 新增场景：`新增归档任务`
   - 编辑场景：`编辑归档任务`
4. 底部固定操作栏：`取消` + `保存并执行`（主按钮）。

## 4.2 表单分区

### 核心信息区（默认可见）

1. 任务名称
2. 源库连接
3. 源库名称
4. 源表名称
5. 归档条件

### 高级配置区（默认展开）

1. 目标库连接（可选）
2. 目标库名称（可选）
3. 目标表名称（可选）
4. 启用状态

注：虽然属于“高级配置”，但本次按用户要求默认展开展示，不折叠隐藏。

## 4.3 提交与反馈

1. 点击“保存并执行”后，按钮进入 loading。
2. 成功后：
   - 关闭抽屉；
   - 列表刷新；
   - 保持成功提示文案明确（如“任务已创建并触发执行”）。
3. 失败后：
   - 抽屉保持打开；
   - 错误提示清晰可见；
   - 保留用户输入，不清空表单。

## 4.4 校验策略

沿用现有校验规则，不改业务含义：

1. 任务名称必填。
2. 源库连接必填。
3. 源库名称必填。
4. 源表名称必填。
5. 归档条件必填。

## 5. 视觉风格规范（企业运维）

1. 视觉基调：简洁、专业、低噪音。
2. 配色：信任蓝 + 中性灰，避免高饱和装饰色。
3. 密度：12-14px 字号体系，紧凑但可读。
4. 层级：
   - 区块标题清晰；
   - 表单标签对齐；
   - 主按钮高对比，次按钮弱化。
5. 动效：仅保留必要过渡，避免花哨动效。

## 6. 实现边界

主要修改文件：

1. `frontend/src/views/ArchiveTaskWithCron.vue`

允许：

1. 将任务表单容器从 `el-dialog` 改为 `el-drawer`。
2. 调整模板结构与样式分区。
3. 优化按钮文案与交互反馈。

不允许：

1. 修改 store 方法签名与 API 协议。
2. 引入与本任务无关的大范围重构。

## 7. 验收标准

1. 点击“新增归档任务”打开右侧抽屉。
2. 抽屉内展示“核心信息 + 高级配置”两个分区。
3. 高级配置默认展开可见。
4. 主操作按钮为“保存并执行”，提交时有 loading。
5. 提交成功后有明确成功反馈且列表状态正确更新。
6. 桌面端与常见窄屏下布局不溢出、不遮挡操作按钮。
